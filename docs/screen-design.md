# Sorter — 화면 기획 문서

macOS 네트워크 라우팅 관리 앱 **Sorter**의 화면 구성 및 상호작용 정의.
관련 문서: [`../CLAUDE.md`](../CLAUDE.md)

---

## 1. 화면 구성 개요

전체는 **사이드바 + 디테일** 형태의 단일 윈도우(`NavigationSplitView`)로 구성한다.

```
┌──────────────────────────────────────────────────────────────┐
│ Sorter                                            [ _ ] [ □ ] [ X ]│
├───────────────┬──────────────────────────────────────────────┤
│  사이드바      │                  메인 영역                     │
│               │                                                │
│ ▸ 디바이스     │   (선택된 메뉴에 따라 화면 전환)               │
│ ▸ 라우트       │                                                │
│               │                                                │
│               │                                                │
├───────────────┴──────────────────────────────────────────────┤
│ 상태바: 권한 상태 · 마지막 동기화 시각 · 메시지                 │
└──────────────────────────────────────────────────────────────┘
```

| 화면 | 역할 | 비고 |
|------|------|------|
| S1. 디바이스 목록 (`DevicesView`) | 네트워크 인터페이스 목록 + 세부 정보 | 읽기 전용 |
| S2. 라우트 목록 (`RoutesView`) | 라우트 전체 목록, 추가/수정/삭제 진입 | 핵심 화면 |
| S3. 라우트 추가/수정 (`RouteEditView`) | 라우트 입력 폼 | 시트(Sheet)로 표시 |
| S4. 삭제 확인 다이얼로그 | 삭제 전 확인 | Alert |
| S5. 권한 요청 흐름 | 관리자 인증 안내 | 시스템 프롬프트 + 안내 |

---

## 2. S1. 디바이스 목록 화면 (`DevicesView`)

현재 시스템에 등록된 네트워크 디바이스를 목록으로 보여주고, 선택 시 세부 정보를 표시한다. **읽기 전용.**

```
┌──────────────────────────────────────────────────────────────┐
│ 네트워크 디바이스                                  [ ⟳ 새로고침 ]│
├──────────────────────────────────────────────────────────────┤
│  상태  표시이름        BSD이름   IPv4              상태         │
│  ●     Wi-Fi          en0       192.168.0.12      up           │
│  ○     Ethernet       en1       —                 down         │
│  ●     Thunderbolt    en2       10.0.0.5          up           │
│  ●     loopback       lo0       127.0.0.1         up           │
├──────────────────────────────────────────────────────────────┤
│  ▼ 세부 정보 (en0 / Wi-Fi)                                     │
│     표시 이름 : Wi-Fi                                          │
│     BSD 이름  : en0                                            │
│     IPv4      : 192.168.0.12                                   │
│     IPv6      : fe80::1c2d...                                  │
│     MAC       : a4:83:e7:xx:xx:xx                              │
│     상태      : up    MTU: 1500                                │
└──────────────────────────────────────────────────────────────┘
```

- **목록 컬럼**: 상태(●up/○down), 표시이름, BSD이름, IPv4, 상태 텍스트.
- **행 선택** 시 하단 세부 정보 패널 갱신.
- **새로고침** 버튼: `NetworkDeviceService` 재조회.
- 자동 새로고침: 화면 진입 시 1회, 이후 수동.
- 빈/에러 상태: "디바이스를 불러올 수 없습니다" 메시지 + 재시도 버튼.

## 3. S2. 라우트 목록 화면 (`RoutesView`) — 핵심

라우트를 한 목록에 보여주되 **앱이 관리하는 라우트**와 **시스템 라우트**를 구분한다.
관리 라우트만 수정/삭제 가능, 시스템 라우트는 잠금 처리.

```
┌──────────────────────────────────────────────────────────────┐
│ 라우트                          [ 필터 ▾ ] [ ⟳ ]  [ + 라우트 추가 ]│
├──────────────────────────────────────────────────────────────┤
│      목적지            프리픽스   인터페이스   유형      액션    │
│ 🔒  default           —          en0          시스템    —      │
│ 🔒  192.168.0.0       /24        en0          시스템    —      │
│ ✎   12.12.3.4         /32        en2          사용자  [수정][삭제]│
│ ✎   10.20.0.0         /16        en1          사용자  [수정][삭제]│
├──────────────────────────────────────────────────────────────┤
│  ▼ 세부 정보 (12.12.3.4/32 → en2)                             │
│     목적지     : 12.12.3.4                                     │
│     프리픽스   : /32 (호스트)                                  │
│     인터페이스 : en2 (Thunderbolt)                            │
│     게이트웨이 : link#... (interface 라우트)                   │
│     등록일     : 2026-06-17 14:20                              │
│     적용 상태  : ✅ 라우팅 테이블에 존재                        │
└──────────────────────────────────────────────────────────────┘
```

### 행 유형 구분
- 🔒 **시스템 라우트**: 회색/잠금 아이콘, 수정·삭제 버튼 없음. (`ManagedRouteStore`에 없는 모든 라우트 + `default`)
- ✎ **사용자 라우트**: `ManagedRouteStore`에 있는 라우트. `[수정] [삭제]` 노출.

### 필터 (드롭다운)
- 전체 / 사용자 라우트만 / 시스템 라우트만.
- 기본값: **전체** (사용자 라우트가 상단 정렬).

### 적용 상태 표시
- ✅ 라우팅 테이블에 실제로 존재 (`netstat -rn` / `route get` 교차 확인).
- ⚠️ 스토어엔 있으나 테이블에 없음 (재적용 버튼 제공).
- 이 교차 검증으로 외부에서 라우트가 지워진 경우를 감지.

### 액션
- **[+ 라우트 추가]** → S3 시트(추가 모드).
- **[수정]** → S3 시트(수정 모드, 기존 값 채움). *사용자 라우트만.*
- **[삭제]** → S4 확인 다이얼로그. *사용자 라우트만.*
- 시스템 라우트 행에서는 수정/삭제 액션이 **렌더링되지 않음**(비활성화가 아니라 미노출).

## 4. S3. 라우트 추가/수정 화면 (`RouteEditView`)

시트(Sheet)로 표시. 추가/수정 공용 폼.

```
┌────────────────────────────────────┐
│  라우트 추가                          │
├────────────────────────────────────┤
│  목적지 유형  ( ) 호스트  (•) 네트워크 │
│                                      │
│  목적지      [ 12.12.3.0          ]  │
│  프리픽스    [ 24 ▾ ]   (/24)        │
│                                      │
│  대상 인터페이스                      │
│      [ en2 — Thunderbolt        ▾ ]  │
│                                      │
│  (선택) 게이트웨이                    │
│      [                          ]    │
│                                      │
│  ┌────────────────────────────────┐ │
│  │ 실행될 명령(미리보기):          │ │
│  │ route -n add -net 12.12.3.0/24 │ │
│  │   -interface en2               │ │
│  └────────────────────────────────┘ │
│                                      │
│  ⚠ 적용 시 관리자 권한이 필요합니다.  │
├────────────────────────────────────┤
│              [ 취소 ]   [ 저장·적용 ] │
└────────────────────────────────────┘
```

### 입력 필드
| 필드 | 설명 | 검증 |
|------|------|------|
| 목적지 유형 | 호스트(/32) / 네트워크(CIDR) | — |
| 목적지 | IPv4/IPv6 주소 | `inet_pton` 유효성 |
| 프리픽스 | 호스트면 자동 /32, 네트워크면 선택 | 0–32(v4)/0–128(v6) |
| 대상 인터페이스 | 디바이스 목록에서 선택 | 존재하는 `bsdName`만 |
| 게이트웨이(선택) | 지정 시 게이트웨이 라우트 | 유효 IP |

### 동작 규칙
- **명령 미리보기**: 입력에 따라 실행될 `route` 명령을 실시간 표시(투명성/안전성).
- 중복 검사: 동일 `destination + prefix` 존재 시 저장 차단 + 안내.
- **수정 모드**: 내부적으로 기존 라우트 `delete` 후 새 값 `add` (CLAUDE.md 정책).
- **[저장·적용]** 클릭 → 권한 흐름(S5) → 성공 시 시트 닫고 목록 갱신, `ManagedRouteStore` 반영.
- 실패 시 시트 유지 + 에러 메시지(명령 stderr를 사용자 친화 문구로 변환).

## 5. S4. 삭제 확인 다이얼로그

```
┌──────────────────────────────────────┐
│  라우트를 삭제할까요?                   │
│                                        │
│  12.12.3.4/32 → en2                    │
│  실행: route -n delete -host 12.12.3.4 │
│                                        │
│  이 작업은 관리자 권한이 필요합니다.    │
│                                        │
│            [ 취소 ]   [ 삭제 ]          │
└──────────────────────────────────────┘
```
- 사용자 라우트에서만 진입 가능.
- 삭제 직전 서비스 레이어에서 `ManagedRouteStore` 포함 여부 재검증(가드).
- 성공 시 스토어·테이블 모두에서 제거, 목록 갱신.

## 6. S5. 권한 요청 흐름

`route add/delete`는 root 권한 필요. (CLAUDE.md 권한 섹션 참조)

```
[저장·적용/삭제] 클릭
      │
      ▼
권한 필요 여부 확인
      │
      ▼
┌─────────────────────────┐
│ 시스템 인증 프롬프트      │  ← MVP: osascript with administrator privileges
│ (관리자 비밀번호/Touch ID)│     배포: XPC 권한 헬퍼(한번 인증 후 생략)
└─────────────────────────┘
      │ 성공                  │ 취소/실패
      ▼                       ▼
명령 실행 → 결과 반영        작업 취소, 변경 없음, 안내 메시지
```

- 상태바에 현재 권한 상태(인증됨/필요) 표시.
- 인증 취소는 에러가 아닌 "변경 없음"으로 처리.

## 7. 공통 UI 규칙

- **상태바(하단)**: 좌측 권한 상태, 우측 마지막 동기화 시각, 중앙 일시 메시지(토스트성).
- **로딩**: 명령 실행 중 해당 액션 버튼 비활성화 + 스피너.
- **에러 표시**: 인라인(폼) 또는 배너(목록). 원본 `stderr`는 접기(▾ 자세히)로 제공.
- **빈 상태**: 사용자 라우트가 없을 때 "추가된 라우트가 없습니다. [+ 라우트 추가]" 안내.
- **접근성**: 모든 액션 버튼에 라벨, 상태 아이콘에 텍스트 대체.

## 8. 화면 전환 흐름도

```
        ┌──────────────┐
        │  앱 시작      │
        └──────┬───────┘
               ▼
     ┌───────────────────┐   사이드바
     │  S2 라우트 목록    │◀────────────▶  S1 디바이스 목록
     └───┬───────┬───────┘
   추가  │       │ 수정/삭제
         ▼       ▼
   ┌──────────┐  ┌──────────────┐
   │ S3 폼    │  │ S4 삭제 확인  │
   └────┬─────┘  └──────┬───────┘
        │ 저장·적용       │ 삭제
        ▼                ▼
   ┌────────────────────────┐
   │  S5 권한 흐름           │
   └──────────┬─────────────┘
              ▼
       목록 갱신 + 스토어 반영
```

## 9. 향후 확장(범위 외, 메모)

- 라우트 일괄 가져오기/내보내기(JSON).
- 프로파일(예: "사무실", "VPN") 단위 라우트 묶음 on/off.
- 변경 이력(audit log) 화면.
